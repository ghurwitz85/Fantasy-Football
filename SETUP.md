# Minimal setup

1. Replace the files in the GitHub repository with this package and commit them to `main`.
2. In **Settings → Pages**, deploy from the `main` branch/root folder.
3. In **Settings → Secrets and variables → Actions**, add:
   - `RANKINGS_URL`: a licensed JSON or CSV rankings endpoint.
   - `RANKINGS_API_KEY`: optional bearer token required by that endpoint.
   - `TEAM_CONTEXT_URL`: optional JSON endpoint containing O-line, QB, defense, and position-specific SOS ranks.
   - `YAHOO_HISTORY_URL`: optional JSON endpoint generated by a secure Yahoo OAuth backend.
4. Open **Actions → Refresh fantasy data → Run workflow** once.

The website works immediately with the bundled cache even before secrets are configured. It refreshes daily at 11:17 UTC after an endpoint is supplied.

## Yahoo security

Never put a Yahoo password, Yahoo client secret, or refresh token in `index.html`. GitHub Pages is public static hosting. Yahoo history can be loaded today with the Historical Performance CSV importer. Automatic Yahoo OAuth requires a small private backend (for example a Cloudflare Worker) and Yahoo application credentials stored as encrypted secrets. The frontend is already prepared to consume the resulting `data/yahoo-history-2025.json` feed.
